  <action>
Modify `lib/hooks/useRemotePrices.ts` to fetch from the Workers API instead of the static `/data/prices.json` file.

Key changes:

1. **Add Workers URL configuration:**

```typescript
const WORKERS_API_URL =
  process.env.NEXT_PUBLIC_WORKERS_URL ?? "https://lenovocompare-prices.YOUR_SUBDOMAIN.workers.dev";
```

Note: The actual subdomain will be known after Plan 03's deployment checkpoint. Use the environment variable approach so the URL is configurable at build time. The hardcoded fallback will be updated with the real URL during Plan 03 execution.

`NEXT_PUBLIC_WORKERS_URL` must be set as a GitHub Actions variable (not secret — it's a public URL) for production builds. For local dev, add to `.env.local` (which is gitignored).

2. **Update fetchPrices():**
   Replace the `${basePath}/data/prices.json` fetch URL with `${WORKERS_API_URL}/api/prices`.

Remove the `getBasePath()` helper function — it's no longer needed since we're fetching from an external URL, not a relative path.

3. **Keep fallback behavior per user decision:**
   "If Workers API is unreachable (cold start, outage, network error), silently fall back to existing seed-prices.ts data bundled in the static build."

The current code already returns `[]` on fetch failure, and `usePrices()` merges remote prices with seed-prices.ts. So the fallback behavior is preserved — when the Workers API fails, remote prices are empty, and seed-prices.ts data fills in.

The existing `console.warn("[useRemotePrices] Fetch failed, falling back to seed prices:", err)` log line stays.

4. **No stale indicator** — per user decision, no error state shown to user. The existing `error` field in the return type can remain for internal debugging but should not be surfaced in any UI.

5. **Keep module-level cache and TTL** — the 15-minute TTL prevents excessive calls to Workers API.

Update the file's doc comment to reflect the new data source.

**Part A: Update deploy.yml for Workers deployment**

Add a Workers deploy job to `.github/workflows/deploy.yml`. This job should run alongside (or after) the static site deploy.

Add a new job `deploy-workers` that runs after the existing deploy job:

```yaml
deploy-workers:
  if: ${{ github.event_name == 'workflow_dispatch' || github.event.workflow_run.conclusion == 'success' }}
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@11bd71901bbe5b1630ceea73d27597364c9af683 # v4.2.2

    - name: Setup Node.js
      uses: actions/setup-node@39370e3970a6d050c480ffad4ff0ed4d3fdee5af # v4.1.0
      with:
        node-version: 20

    - name: Install Workers dependencies
      run: cd workers && npm ci

    - name: Apply D1 Migrations
      uses: cloudflare/wrangler-action@v3
      with:
        apiToken: ${{ secrets.CLOUDFLARE_API_TOKEN }}
        accountId: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}
        workingDirectory: workers
        command: d1 migrations apply lenovocompare-prices --remote --yes

    - name: Deploy Workers
      uses: cloudflare/wrangler-action@v3
      with:
        apiToken: ${{ secrets.CLOUDFLARE_API_TOKEN }}
        accountId: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}
        workingDirectory: workers
```

Also update the existing `deploy` job's Build step to include the Workers URL environment variable:

```yaml
- name: Build
  run: npm run build
  env:
    NEXT_PUBLIC_WORKERS_URL: ${{ vars.NEXT_PUBLIC_WORKERS_URL }}
```

Note: `vars.NEXT_PUBLIC_WORKERS_URL` is a GitHub Actions variable (not secret). The user needs to set this in the repository settings: Settings -> Secrets and variables -> Actions -> Variables -> New repository variable.

**Part B: Create Playwright smoke config**

Create `playwright.smoke.config.ts` as a SEPARATE config from the existing `playwright.config.ts` (which targets the dev server). This config targets the static `out/` export.

```typescript
import { defineConfig } from "@playwright/test";

export default defineConfig({
  testDir: "./tests/e2e",
  testMatch: "smoke.spec.ts",
  fullyParallel: true,
  retries: 0,
  workers: 1,
  reporter: "list",
  use: {
    baseURL: "http://localhost:3001",
    trace: "off",
  },
  webServer: {
    command: "npx serve out -p 3001 -s",
    port: 3001,
    reuseExistingServer: true,
    timeout: 30000,
  },
});
```

The `-s` flag enables single-page app mode for client-side routing. `serve` is used via npx (no global install needed).

test.describe("Static export smoke tests", () => {
  test("home page loads with model cards", async ({ page }) => {
    await page.goto("/");
    // Wait for content to render
    await expect(page.locator("h1")).toBeVisible();
    // Check at least one model card renders
    const cards = page.locator('[class*="carbon-card"]');
    await expect(cards.first()).toBeVisible({ timeout: 10000 });
  });

  test("compare page renders without errors", async ({ page }) => {
    await page.goto("/compare");
    await expect(page.locator("h1")).toContainText(/Compare/i);
  });

  test("model detail page renders", async ({ page }) => {
    await page.goto("/model/x1-carbon-gen12");

  });

  test("pricing page renders", async ({ page }) => {

  });

});
