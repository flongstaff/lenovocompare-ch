# Technology Stack

**Analysis Date:** 2025-03-13

## Languages

**Primary:**

- TypeScript 5 - All application code, type-safe across frontend and build scripts
- CSS/Tailwind CSS 3 - Styling with custom Carbon Design System dark theme
- JavaScript/JSX/TSX - React components via TypeScript

**Secondary:**

- Bash/Shell - Docker and deployment scripts
- NGINX Configuration - Production reverse proxy configuration in `nginx.conf`

## Runtime

**Environment:**

- Node.js >=20 (specified in `package.json` engines)

**Package Manager:**

- npm with npm CI for reproducible builds
- Lockfile: `package-lock.json` present and required

**Build Tools:**

- Next.js 16.1.6 (App Router, static export)
- tsx 4.21.0 - TypeScript script executor for build/validation scripts
- PostCSS 8 - CSS processing pipeline for Tailwind

## Frameworks

**Core:**

- Next.js 16.1.6 - React framework with App Router, static site generation (output: "export" in `next.config.mjs`)
- React 18 - UI library
- React DOM 18 - DOM rendering

**Styling & Fonts:**

- Tailwind CSS 3.4.1 - Utility-first CSS framework
- Geist 1.7.0 - Font package (GeistSans and GeistMono variable fonts)

**Charting & Visualization:**

- jsdom 28.1.0 - DOM implementation for Node.js test environment
- @playwright/test 1.58.2 - E2E testing framework

**Build & Dev:**

- @next/bundle-analyzer 16.1.6 - Bundle size analysis (optional via ANALYZE env var)
- ESLint 9 with next/core-web-vitals and next/typescript configurations
- Prettier 3.8.1 - Code formatter with custom config

## Key Dependencies

**Critical Production:**

- recharts 3.7.0 - Why it matters: chart rendering for performance/price comparisons; bundle cost ~324 KB is significant
- geist 1.7.0 - Custom typography with variable fonts for branding

**Build & Validation:**

- tsx 4.21.0 - Enables TypeScript script execution for data validation and price generation scripts
- playwright 1.58.2 - Browser automation for PSREF scraping and E2E testing

**Type Safety:**

- @types/node 20 - Node.js type definitions
- @types/react 18 - React type definitions
- @types/react-dom 18 - React DOM type definitions

## Configuration

**Environment:**

- `.env` files absent - project is fully local-first with no secrets or API keys required
- Environment variables used:
  - `ANALYZE=true` - Enables bundle analyzer on build
  - `GITHUB_ACTIONS=true` - GitHub Actions detection for basePath configuration (sets `/lenovocompare-ch` prefix for GitHub Pages deployment)
  - `NODE_ENV` - Implicit Next.js setting (development/production)

**Build:**

- `next.config.mjs` - Static export configuration, bundle analyzer integration, unoptimized images (static-only)
- `tsconfig.json` - Strict TypeScript mode, ES2017 target, path aliases (`@/*` → project root)
- `tailwind.config.ts` - Custom Carbon dark theme colors, animation keyframes
- `.eslintrc.json` - Next.js core web vitals + TypeScript rules
- `.prettierrc.json` - 2-space indentation, double quotes, trailing commas, 120-char line width, Tailwind plugin
- `postcss.config.mjs` - Tailwind CSS processing
- `vitest.config.ts` - Unit test configuration with jsdom environment
- `playwright.config.ts` - E2E test configuration targeting localhost:3000

## Platform Requirements

**Development:**

- Node.js 20+
- npm 10+ (for npm ci)
- Docker (optional, for containerized dev via `docker compose up`)

**Production:**

- Deployment target: GitHub Pages (via GitHub Actions)
- Static hosting only (no backend server required)
- NGINX reverse proxy for production Docker deployment
- Basepath configuration for GitHub Pages deployment at `/lenovocompare-ch/`

## Output & Artifacts

**Static Export:**

- Build output: `out/` directory (generated by Next.js with `output: "export"`)
- Deployed to GitHub Pages or NGINX-served static assets
- Docker production image: NGINX serving `out/` on port 3000

**Generated Data:**

- `/data/prices.json` - Generated at build time from seed prices + community contributions (via `npm run generate-prices`)
- Type-checked validation scripts run during build to ensure data integrity
