## Data Storage

**Databases:**

- None - The application is fully local-first with no backend database.

**File Storage:**

- Local filesystem only - No cloud storage integration.

**Client-Side Storage:**

- localStorage - User-contributed prices and price management
  - Keys: `lenovocompare-prices` (current), legacy `thinkcompare-prices` (migrated)
  - Managed by `lib/hooks/useLocalStorage.ts` with automatic migration
- sessionStorage - Compare selection state (temporary per-session)
  - Key: `lenovocompare-compare`
  - Managed by `lib/hooks/useCompare.ts`
  - Clears on tab close/new session (not persistent across tabs)

**Caching:**

- Module-level cache in `lib/hooks/useRemotePrices.ts` - 15-minute TTL for `/data/prices.json` fetch
- No external cache service (Redis, Memcached, etc.)

## Data Sources

**Hardcoded Static Data:**

- Laptop specs: `data/laptops.ts` - 124 models from public Lenovo PSREF data (2018–2025)
- CPU benchmarks: `data/cpu-benchmarks.ts` - Composite scores (0-100) + raw Cinebench 2024/Geekbench 6 data
- GPU benchmarks: `data/gpu-benchmarks.ts` - 28+ GPU scores/gaming tiers with FPS estimates
- Model benchmarks: `data/model-benchmarks.ts` - Per-model chassis data (thermals, noise, battery, display)
- Linux compatibility: `data/linux-compat.ts` - Certified distros and driver notes for all models
- Editorial content: `data/model-editorial.ts` - Curated text for all 124 models
- Hardware guide: `data/hardware-guide.ts` - CPU/GPU analysis and recommendations
- Seed prices: `data/seed-prices.ts` - ~205 curated Swiss retail prices (CHF) covering all models
- Price baselines: `data/price-baselines.ts` - Static MSRP and historical low prices for all models

**Build-Time Generated Data:**

- `/data/prices.json` - Merged seed prices + community contributions (generated by `npm run generate-prices`)
  - Generated from `scripts/generate-prices-json.ts`
  - Includes seed prices and community contributions
  - Fetched by `useRemotePrices` hook at 15-minute cache interval

## Data Ingestion

**PSREF Scraping (Development Tool):**

- `npm run scrape-psref` - Playwright-based scraper for Lenovo PSREF specifications
  - Extracts processor, display, memory, storage, GPU options from PSREF Specifications tab
  - Script: `scripts/scrape-psref.ts`
  - No ongoing API calls; manual data collection tool for developers
  - Results merged via `npm run merge-psref` into `data/laptops.ts`

**Community Price Submissions:**

- GitHub Issues API integration (GitHub Actions automation)
- `.github/workflows/process-price-submission.yml` - Automated PR creation on `price-approved` label
- Parses issue body for: laptop model ID, retailer, price (CHF), price type, product URL
- Converts to `SwissPrice` object and proposes pull request for manual review

**No Ongoing Data Feeds:** All data is static post-build. Updates require code commits or manual GitHub issue workflow.

## CI/CD & Deployment

**Hosting:**

- Primary: GitHub Pages (static deployment)
- Secondary: Docker + NGINX for self-hosted static serving
- Deployment target: `out/` directory (Next.js static export)

**CI Pipeline:**

- Service: GitHub Actions
- Workflows:
  - `.github/workflows/ci.yml` - Lint (ESLint) + Test (Vitest) + Build (Next.js)
  - `.github/workflows/deploy.yml` - Build + Deploy to GitHub Pages on main branch
  - `.github/workflows/codeql.yml` - CodeQL security analysis
  - `.github/workflows/process-price-submission.yml` - Community price PR automation

**Build Process:**

1. `npm run validate` - Data integrity checks via `scripts/validate-data.ts`
2. `npm run generate-prices` - Build `/data/prices.json` from seed + community data
3. `npm run build` - Next.js static export to `out/`

## Environment Configuration

**Optional env vars:**

- `GITHUB_ACTIONS=true` - Auto-set by GitHub Actions; enables basePath for GitHub Pages

**Secrets location:** None - No secrets stored. No `.env` files required.

**Configuration Management:**

- All configuration is in TypeScript/JSON config files (committed to git)
- No environment-dependent configuration beyond basePath for GitHub Pages

## Build Artifacts & Cache

**Generated Files:**

- `.next/` - Next.js build cache (auto-cleared on file delete/move)
- `out/` - Static export output (deployed)

**Docker Image:**

- Build: `node:20-alpine` → compile to static export

**Community Feedback Loop:**
